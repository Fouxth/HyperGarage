## 1. การวิเคราะห์ระบบ (System Analysis)

การวิเคราะห์ระบบ HyperGarage มุ่งเน้นการแก้ปัญหาความยุ่งยากในการเลือกซื้ออะไหล่แต่งรถยนต์ที่ต้องเข้ากันได้กับรุ่นรถของลูกค้า (Vehicle Compatibility) และการจัดการหลังบ้านที่ยังไม่เป็นระบบ

### 1.1 ความต้องการด้านฟังก์ชันการทำงานหลัก (Functional Requirements)

**กลุ่มที่ 1: ลูกค้า (Customer Frontend)**

- **Compatibility Checker (ระบบตรวจสอบความเข้ากันได้):** สามารถเลือกยี่ห้อรถ → รุ่น → เจเนอเรชัน → รหัสเครื่องยนต์ แล้วระบบจะแสดงเฉพาะอะไหล่ที่เข้ากันได้กับรถคันนั้นเท่านั้น ช่วยป้องกันลูกค้าซื้ออะไหล่ผิดรุ่น
- **Guest Checkout (สั่งซื้อโดยไม่ต้องสมัครสมาชิก):** ลูกค้าสามารถเพิ่มสินค้าลงตะกร้า กรอกข้อมูลจัดส่ง เลือกวิธีชำระเงิน และยืนยันคำสั่งซื้อได้โดยไม่ต้องสร้างบัญชีผู้ใช้ ระบบจะตัดสต็อกสินค้าทันทีหลังสั่งซื้อสำเร็จ
- **Order Tracking (ติดตามคำสั่งซื้อ):** ลูกค้าสามารถติดตามสถานะคำสั่งซื้อได้แบบเรียลไทม์ผ่านหมายเลขเบอร์โทรศัพท์ (รอดำเนินการ → ยืนยันแล้ว → จัดส่งแล้ว)
- **Product Reviews (รีวิวสินค้า):** ลูกค้าสามารถเขียนรีวิวสินค้า ให้คะแนน 1-5 ดาว พร้อมความคิดเห็น โดยระบบจะคำนวณคะแนนเฉลี่ยแบบ real-time
- **Flash Sale & Promotions (โปรโมชัน):** ดูสินค้าลดราคา Flash Sale และใช้คูปองส่วนลดในการสั่งซื้อ ระบบรองรับทั้งคูปองแบบเปอร์เซ็นต์และแบบจำนวนเงินคงที่

**กลุ่มที่ 2: ผู้ดูแลระบบ (Admin Back-office)**

- **Product CRUD (จัดการสินค้า):** เพิ่ม แก้ไข ลบสินค้า หมวดหมู่ และแบรนด์ต่างๆ ได้ ระบบมีการป้องกันไม่ให้ลบสินค้าที่มีคำสั่งซื้อแล้ว (ตอบ 409 Conflict)
- **Inventory Management (จัดการสต็อก):** ดูสต็อกสินค้าทั้งหมด มีการแจ้งเตือนสินค้าใกล้หมด (Low Stock Alert) พร้อมแสดงจำนวนคงเหลือแบบ real-time
- **Vehicle Taxonomy (จัดการข้อมูลรถยนต์):** จัดการข้อมูลยี่ห้อรถ → รุ่น → เจเนอเรชัน → รหัสเครื่องยนต์ และผูกความเข้ากันได้กับสินค้าแต่ละชิ้น
- **Order Management (จัดการคำสั่งซื้อ):** ดูรายการคำสั่งซื้อทั้งหมด อัปเดตสถานะ (pending → confirmed → shipped → delivered) ดูรายละเอียดออเดอร์และข้อมูลจัดส่ง
- **Coupon Management (จัดการคูปอง):** สร้าง แก้ไข เปิด/ปิดคูปองส่วนลด กำหนดเงื่อนไขใช้งาน (ยอดขั้นต่ำ, จำนวนครั้ง, วันหมดอายุ)
- **Reports & Dashboard (รายงานสรุป):** แดชบอร์ดแสดงยอดขายรวม จำนวนคำสั่งซื้อ สินค้าขายดี หมวดหมู่ยอดนิยม และกราฟแนวโน้ม
- **Store Settings (ตั้งค่าร้าน):** ตั้งค่าชื่อร้าน ข้อมูลติดต่อ ค่าจัดส่ง และเปิด/ปิด Maintenance Mode

### 1.2 ความต้องการด้านที่ไม่ใช่ฟังก์ชัน (Non-Functional Requirements)

- **Performance:** API Response Time ≤ 500ms (p95), รองรับผู้ใช้พร้อมกัน ≥ 100 คน
- **Usability:** Responsive Design รองรับทุกขนาดจอ (Desktop, Tablet, Mobile), รองรับ Light/Dark Theme
- **Security:** ใช้ Prisma ORM ป้องกัน SQL Injection, React auto-escaping ป้องกัน XSS, Input Validation ทุก endpoint
- **Reliability:** ระบบ Atomic Transaction สำหรับการสั่งซื้อ ป้องกันสต็อกติดลบ
